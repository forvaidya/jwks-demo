# AWS STS OIDC Integration Implementation Summary

## Overview

I've implemented a complete passwordless AWS credential acquisition system using OIDC federation. The system consists of:

1. **JWKS Server** - FastAPI server that generates and serves RSA public keys
2. **Private Key Storage** - Securely saves private keys for token signing
3. **AWS STS Integration** - Python script to fetch credentials via OIDC

## What Was Implemented

### 1. Enhanced JWKS Generator (`jwks_generator.py`)

**Changes:**
- Modified `generate_jwks()` to return both JWKS dictionary and private key PEM
- Added `private_key_to_pem()` function to convert RSA key to PEM format
- Added `save_private_key()` function to persist private key to disk
- Returns: `Tuple[Dict, str]` containing JWKS and private key PEM

**Key Functions:**
```python
jwks, private_key_pem = generate_jwks(kid="default")
save_private_key(private_key_pem, "./private_key.pem")
```

### 2. FastAPI Server Updates (`main.py`)

**Changes:**
- Modified lifespan context manager to store private key
- Saves private key to `./private_key.pem` with permissions 600 (owner read-only)
- Added success message confirming key generation

**Behavior:**
- On startup: Generates new RSA keypair, saves private key, caches JWKS
- Serves JWKS at `GET /.well-known/jwks.json`
- Private key persists for use by AWS STS script

### 3. AWS STS Integration (`aws_sts.py`) - Complete Rewrite

**Major Changes:**

#### Old Issues Fixed:
- ❌ Used "dummy-token" → ✅ Creates properly signed JWT
- ❌ Never called JWT creation code → ✅ Uses `create_oidc_jwt()` properly
- ❌ Assumed existing credentials → ✅ Loads from private key file

#### New Functions:

**`load_private_key_from_file(key_path)`**
- Loads private key from disk (generated by server)
- Raises helpful error if key not found
- Used by all credential functions

**`create_oidc_jwt(private_key_pem, user_id, issuer, kid)`**
- Creates JWT token signed with RS256
- Payload includes: `iss`, `sub`, `aud`, `iat`, `exp`
- Headers include: `kid` (key ID)
- Token expires in 1 hour

**`assume_role_with_web_identity_oidc(role_arn, web_identity_token, ...)`**
- Calls AWS STS `AssumeRoleWithWebIdentity` with signed JWT
- Returns credentials: AccessKeyId, SecretAccessKey, SessionToken, Expiration

**`get_aws_credentials(region, private_key_path, issuer)`**
- Constructs proper role ARN from `MAHESH_AWS_ROLE` env var
- Loads private key from disk
- Creates signed JWT token
- Calls AWS STS to assume role
- Returns temporary credentials

**`save_private_key(private_key_pem, key_path)`**
- Saves private key with secure permissions (600)
- Used by both generator and server

#### Environment Variables:
```
MAHESH_AWS_ROLE = "123456789012:role/MyRole"  # Account ID and role name
AWS_USER_ID = "user@example.com"               # User identifier for JWT
```

#### Role ARN Format Support:
- Short format: `123456789012:role/MyRole` → Converts to full ARN
- Full ARN: `arn:aws:iam::123456789012:role/MyRole` → Used as-is

## Architecture Flow

### Startup Flow
```
FastAPI Server starts
    ↓
Generates RSA keypair (2048-bit)
    ↓
Saves private key to ./private_key.pem (chmod 600)
    ↓
Serves public key as JWKS at /.well-known/jwks.json
```

### AWS Credential Flow
```
User runs: python3 aws_sts.py
    ↓
Loads MAHESH_AWS_ROLE and AWS_USER_ID from env
    ↓
Loads private key from ./private_key.pem
    ↓
Creates JWT with RS256 signature:
  - iss: http://localhost:8000 (or Argo Tunnel URL)
  - sub: AWS_USER_ID
  - aud: sts.amazonaws.com
  - exp: now + 1 hour
    ↓
Calls AWS STS AssumeRoleWithWebIdentity with JWT
    ↓
AWS validates JWT against JWKS endpoint
    ↓
Returns temporary credentials (valid for 1 hour)
```

## Security Features

✅ **Private Key Protection**
- Stored with 0o600 permissions (owner read-only)
- Never committed to git (in .gitignore)
- Regenerated on each server restart

✅ **Token Signing**
- Uses RSA-2048 with RS256 algorithm
- Tokens include expiration (1 hour)
- Signed with private key only

✅ **JWT Claims**
- Standard claims: iss, sub, aud, iat, exp
- Custom kid (key ID) in header
- AWS validates against JWKS

✅ **AWS Credential Handling**
- Credentials only in memory (not saved to disk)
- SessionToken provided by AWS STS (temporary)
- Credentials logged with redaction in examples

## Usage Examples

### Basic Usage
```bash
# Terminal 1: Start server
source venv/bin/activate
python3 main.py

# Terminal 2: Get AWS credentials
export MAHESH_AWS_ROLE="123456789012:role/MyRole"
export AWS_USER_ID="user@example.com"
python3 aws_sts.py
```

### With Argo Tunnel
```bash
# Terminal 1: Start server
python3 main.py

# Terminal 2: Expose via tunnel
cloudflared tunnel run --url http://localhost:8000

# Terminal 3: Get credentials (using tunnel URL as issuer)
# Credentials obtained via AWS are valid for 1 hour
```

### Programmatic Usage
```python
from aws_sts import get_aws_credentials, set_aws_env_from_credentials
import os

os.environ["MAHESH_AWS_ROLE"] = "123456789012:role/MyRole"
os.environ["AWS_USER_ID"] = "user@example.com"

# Get credentials
creds = get_aws_credentials()

# Use them
set_aws_env_from_credentials(creds)

# Or read directly
print(creds["AccessKeyId"])
print(creds["SecretAccessKey"])
print(creds["SessionToken"])
print(creds["Expiration"])
```

## Important Notes

⚠️ **Private Key Security**
- Keep `./private_key.pem` confidential
- Never commit to git (already in .gitignore)
- Regenerated on each server restart
- Use filesystem permissions to restrict access

⚠️ **Token Validity**
- JWT tokens expire in 1 hour
- AWS credentials typically valid for 1 hour (configurable)
- Run `aws_sts.py` again to get fresh credentials

⚠️ **AWS Configuration Required**
- Must set up OIDC provider in AWS IAM
- Must create role with OIDC trust policy
- Must configure proper audience (`sts.amazonaws.com`)

## Troubleshooting Reference

| Error | Cause | Solution |
|-------|-------|----------|
| "Private key not found" | Server not started | Run `python3 main.py` first |
| "InvalidIdentityToken" | Expired JWT or wrong issuer | Run `aws_sts.py` to get fresh token |
| "AssumeRoleWithWebIdentity not authorized" | Missing/wrong trust policy | Check AWS IAM role configuration |
| "OIDC provider not found" | Provider not created in AWS | Create OIDC provider in AWS IAM |

For more details, see AWS_SETUP.md.
